# Sprint 1 — Authentication

## 1. Цель спринта
Реализовать базовую аутентификацию пользователей:
- регистрация
- вход в систему
- хранение пользователей в базе данных

---

## 2. User Stories
- US-1 Регистрация
- US-2 Вход в систему

---

## 3. Объём работ

### 3.1 Backend
- модель User в базе данных
- создание пользователя (регистрация)
- проверка данных при входе
- хеширование пароля

### 3.2 Frontend
- страница регистрации
- страница входа
- формы ввода (email, password)
- обработка ошибок

### 3.3 Инфраструктура
- подключение базы данных
- настройка ORM (Prisma)

---

## 4. Критерии готовности (Definition of Done)
- пользователь может зарегистрироваться
- пользователь может войти в систему
- пароль хранится в зашифрованном виде
- ошибки обрабатываются корректно
- реализованы базовые unit-тесты для backend-логики
- acceptance criteria проверены вручную
- код проходит CI (lint + build)

---

## 5. Ограничения
В рамках Sprint 1:
- не реализуется восстановление пароля
- не добавляются роли в интерфейс
- не реализуется сложная авторизация
