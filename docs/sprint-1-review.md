# Sprint 1 Review — Authentication

## 1. Цель спринта
Реализовать базовую аутентификацию пользователей:
- регистрация
- вход в систему
- хранение пользователей в базе данных

---

## 2. Что было сделано

### Backend
- реализован endpoint регистрации (`/api/auth/register`)
- реализован endpoint входа (`/api/auth/login`)
- добавлено хеширование паролей (bcrypt)
- реализована обработка ошибок

### Frontend
- создана страница `/register`
- создана страница `/login`
- реализованы формы и отправка запросов
- отображение ошибок и успешных действий

### Тестирование
- написаны unit-тесты для `registerUser`
- проверены позитивные сценарии
- проверены негативные сценарии
- проведено ручное тестирование API и UI

---

## 3. Результат спринта

Пользователь может:
- зарегистрироваться
- войти в систему

Система корректно обрабатывает:
- повторную регистрацию
- неверные учетные данные

---

## 4. Критерии готовности

Все критерии Definition of Done выполнены:
- функциональность реализована
- ошибки обрабатываются
- unit-тесты присутствуют
- код проходит сборку и lint

---

## 5. Вывод

Sprint 1 завершён успешно.

Реализована базовая аутентификация, которая является фундаментом для дальнейшей разработки системы заявок.
