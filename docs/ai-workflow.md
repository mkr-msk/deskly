# AI Workflow — Deskly

## 1. Цель
Использовать AI-агента (Codex) как вспомогательный инструмент для разработки,
при этом сохраняя контроль над архитектурой, требованиями и качеством кода.

---

## 2. Роли

### Человек (разработчик)
- принимает все архитектурные решения
- формирует задачи
- проверяет результат
- отвечает за финальное качество

### AI (Codex)
- реализует отдельные задачи
- пишет код
- предлагает решения
- помогает с рефакторингом и исправлением ошибок

---

## 3. Принцип работы

Каждая задача проходит цикл:

1. Формулируется задача (на основе user stories / backlog)
2. Задача упрощается до минимального объёма
3. Задача передаётся Codex
4. Результат проверяется человеком
5. При необходимости дорабатывается
6. Коммит в репозиторий

---

## 4. Ограничения

- AI не принимает архитектурные решения самостоятельно
- AI не изменяет код вне поставленной задачи
- каждая задача должна быть атомарной
- результат всегда проверяется вручную

---

## 5. Использование в проекте

AI используется для:
- написания backend-логики
- создания API
- написания тестов
- генерации boilerplate-кода

AI не используется для:
- принятия продуктовых решений
- изменения структуры проекта без обсуждения
