# Frontend Pages — Deskly (Sprint 1)

## 1. Цель
Определить минимальный набор страниц для реализации аутентификации.

---

## 2. Страницы

### /register
Страница регистрации пользователя.

Функциональность:
- ввод email
- ввод пароля
- отправка формы
- отображение ошибок

---

### /login
Страница входа в систему.

Функциональность:
- ввод email
- ввод пароля
- отправка формы
- отображение ошибок

---

## 3. Ограничения
- без дизайна и стилизации (минимальный UI)
- без редиректов после входа (пока)
- без хранения сессии (пока)
