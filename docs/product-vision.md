# Product Vision — Deskly

## 1. Название продукта
Deskly

## 2. Краткое описание
Deskly — это простой веб-сервис для учёта и обработки рабочих заявок внутри небольшой команды или организации.

Пользователи могут создавать заявки, отслеживать их статус и общаться по ним в комментариях.
Исполнители могут просматривать заявки, менять их статус и отвечать пользователям.
Администратор имеет полный доступ к системе и управлению пользователями.

## 3. Проблема
Во многих небольших командах рабочие заявки обрабатываются хаотично: в мессенджерах, устно или в разных несвязанных инструментах.
Из-за этого заявки теряются, их статусы непонятны, а история обсуждения не сохраняется в одном месте.

## 4. Решение
Deskly предоставляет единое место для регистрации, просмотра и сопровождения заявок.
Система позволяет:
- создавать заявки;
- отслеживать их состояние;
- вести обсуждение в комментариях;
- разделять права доступа по ролям.

## 5. Целевая аудитория
Небольшие команды и организации, которым нужен простой универсальный трекер внутренних рабочих заявок без сложной CRM-логики и перегруженного функционала.

## 6. Основные роли
### User
Создаёт заявки, просматривает свои заявки, пишет комментарии.

### Agent
Просматривает заявки, меняет статус, отвечает в комментариях.

### Admin
Имеет полный доступ к системе и управляет пользователями.

## 7. Ценность продукта
Deskly делает работу с заявками прозрачной, понятной и централизованной.
Все участники процесса видят актуальное состояние заявки и историю взаимодействия в одном месте.

## 8. Границы первой версии
Первая версия продукта должна быть простой и включать только базовый набор возможностей, необходимых для работы с заявками.
Пользователь может создать заявку, исполнитель может её обработать, а администратор может управлять доступом и пользователями внутри одной работающей системы.
