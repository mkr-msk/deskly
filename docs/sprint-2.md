# Sprint 2 — Tickets (Create & List)

## 1. Цель спринта
Реализовать базовую работу с заявками:
- создание заявки
- просмотр списка заявок

---

## 2. User Stories
- US-3 Создание заявки
- US-4 Просмотр списка заявок

---

## 3. Объём работ

### Backend
- модель Ticket в базе данных
- создание заявки (API)
- получение списка заявок пользователя

### Frontend
- форма создания заявки
- страница списка заявок

### Инфраструктура
- обновление Prisma schema
- миграция базы данных

---

## 4. Критерии готовности (Definition of Done)
- пользователь может создать заявку
- пользователь видит список своих заявок
- ошибки обрабатываются корректно
- реализованы базовые unit-тесты
- acceptance criteria проверены вручную
- код проходит CI (lint + build)

---

## 5. Ограничения
- без комментариев
- без назначений исполнителей
- без фильтров
- без статусов кроме базового
